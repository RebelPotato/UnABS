Abstract machines

Abstract machines are powerful ways of executing programs. An abstract machine consists of:

  • a function to transform a program into a "state"
  • a function turning one state to another (or get stuck on an end state)
  • and another function to extract a return value from an end state

Executing a program is straight-forward. Transform the program into an initial state, step through the states until we cannot go any further, and extract the return value from the final state.

Having an abstract machine for a language gives you the ability to execute step by step, essential for understanding a crazy language with call/cc and stuff.
